# Обучение модели играть в Pac‑Man (Atari)

Проект демонстрирует обучение простой модели играть в **Pac‑Man** из набора игр Atari с использованием библиотек Gymnasium и PyTorch. Реализованы стратегии случайного поиска и восхождения на вершину для поиска оптимальных параметров линейной политики.

## Запуск в Google Colab

Чтобы запустить код в Google Colab:

1. Откройте [Google Colab](https://colab.research.google.com/).
2. Создайте новый ноутбук или откройте существующий.
3. Загрузите код проекта одним из способов:

    * **Способ 1.** Скопируйте содержимое файла `.ipynb` и вставьте в ячейки ноутбука.
    * **Способ 2.** Загрузите файл `.ipynb` с локального компьютера:
        * в меню Colab выберите **Файл → Загрузить ноутбук**;
        * выберите файл с кодом.
    * **Способ 3.** Откройте напрямую из GitHub:
        * в меню Colab выберите **Файл → Открыть блокнот**;
        * перейдите на вкладку **GitHub**;
        * вставьте URL репозитория и выберите нужный файл `.ipynb`.

4. Выполните ячейку с установкой зависимостей:

    ```bash
    !pip install "gymnasium[atari,recording]"
    !pip install autorom[accept-rom-license]
    !apt-get update && apt-get install -y xvfb ffmpeg
    ```

5. Последовательно запустите остальные ячейки кода:
    * импорт библиотек и регистрация окружения Atari;
    * создание среды Pac‑Man;
    * предобработка изображений;
    * реализация стратегий обучения (случайный поиск, восхождение на вершину);
    * финальная оценка лучшей модели и запись игрового эпизода.

6. Дождитесь завершения обучения и просмотра результатов: графиков и видео.

## Достигнутые результаты

В ходе выполнения проекта удалось:

1. **Настроить окружение Atari** в Google Colab, включая:
    * загрузку ROM‑файлов через AutoROM;
    * установку системных пакетов (`xvfb`, `ffmpeg`) для рендеринга и записи видео.

2. **Реализовать предобработку состояния**:
    * преобразование RGB‑изображений в оттенки серого;
    * изменение размера кадра до $84 \times 84$ пикселей;
    * нормализация значений пикселей (деление на 255).

3. **Обучить модель с помощью стратегии случайного поиска**:
    * проведено 200 эпизодов случайного подбора весовых параметров;
    * найдено начальное «лучшее» решение с вознаграждением **1650.0**.

4. **Улучшить модель с помощью восхождения на вершину**:
    * выполнено 100 итераций с добавлением шума к весам;
    * достигнуто улучшение вознаграждения до **1680.0** на двух итерациях;
    * продемонстрирована ограниченность метода: большинство итераций не дали улучшения.

5. **Записать и визуализировать игровой процесс**:
    * записан лучший эпизод с максимальным вознаграждением;
    * создано и оптимизировано видео игрового процесса для просмотра в браузере.

6. **Проанализировать эффективность стратегий**:
    * случайный поиск нашёл приемлемое решение за счёт большого числа проб;
    * восхождение на вершину позволило незначительно улучшить результат, но столкнулось с проблемой локальных оптимумов;
    * показано, что простые стратегии поиска могут давать осмысленные результаты даже без сложных алгоритмов RL.

## Выводы

Проведённый эксперимент подтвердил, что даже простые методы оптимизации (случайный поиск и восхождение на вершину) способны обучить базовую линейную модель играть в сложную игру Atari, такую как Pac‑Man.

Несмотря на скромные результаты по сравнению с современными методами RL, подход демонстрирует:
* важность предобработки визуальных данных;
* влияние выбора стратегии оптимизации на итоговую производительность;
* необходимость более продвинутых методов (например, градиента политики или DQN) для достижения высоких результатов.
